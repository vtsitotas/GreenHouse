## How a unit is built (current, working â€” NM-native)

1. **Pi Imager:** Raspberry Pi OS Lite **32-bit** (Pi Zero W = ARMv6). Edit Settings: hostname `greenhouse`, SSH on, user `pi`/`greenhouse2026`, WiFi *with internet* + country `GR`, locale `Europe/Athens`.
2. **Install from PC:**
   ```powershell
   $pi = "C:\Users\billy\Desktop\diplomatikh\pi"
   ssh pi@greenhouse.local "mkdir -p /home/pi/greenhouse"
   scp -r "$pi\scripts" "$pi\systemd" "$pi\portal" "$pi\mosquitto" "$pi\install.sh" pi@greenhouse.local:/home/pi/greenhouse/
   ssh pi@greenhouse.local 'find /home/pi/greenhouse -type f \( -name "*.sh" -o -name "*.py" -o -name "*.service" -o -name "*.conf" \) -exec sed -i "s/\r$//" {} +; sudo bash /home/pi/greenhouse/install.sh && sudo bash /home/pi/greenhouse/scripts/selftest.sh'
   ```
   Expect `RESULT: 12 passed, 0 failed`.
3. **Make the golden image (once):** `ssh pi@greenhouse.local "sudo systemd-run --collect --unit=prep bash /home/pi/greenhouse/scripts/prep_image.sh"` â†’ powers off â†’ pull SD â†’ read to `greenhouse.img` with Win32DiskImager.
4. **Mass-produce:** flash `greenhouse.img` to any SD â†’ boots ready; each unit self-generates MQTT password + `Greenhouse-XXXX` SSID (and, after the plan runs, its own TLS certs + OS password).

> NOTE: `git clone` from the GitHub repo can replace the scp+CRLF step on the Pi once the repo is the source of truth â€” cleaner. (CRLF strip kept for now; `.gitattributes` is a pending nicety.)
